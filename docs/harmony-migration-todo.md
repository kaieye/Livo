# Livo Harmony Migration Todo

目标：将桌面端的核心 RSS 阅读能力按优先级迁移到 HarmonyOS NEXT，先建立稳定的数据与阅读闭环，再逐步补齐搜索、AI、发现和高级媒体能力，避免一开始就把桌面端概念硬搬到移动端。

当前判断：`apps/harmony` 已具备原生工程骨架、基础页面、设置持久化和一条简单 RSS 拉取链路，但仍处于演示阶段，尚未进入正式功能迁移。

## 推荐执行顺序

1. 先完成 Phase 1，锁定迁移边界和首期范围。
2. 再完成 Phase 2 和 Phase 3，把正式数据层与订阅同步链路建起来。
3. 然后完成 Phase 4，确保 Harmony 端具备独立可用的阅读闭环。
4. 最后再做 Phase 5 之后的增强能力，避免过早分散精力。
