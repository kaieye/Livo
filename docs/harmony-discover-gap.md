# Harmony Discover 与 Windows 桌面端差距清单

## 当前已具备

- 支持输入 RSS / Atom 地址并预览
- 支持 `rsshub://` 协议输入
- 支持部分平台主页链接解析
- 支持部分关键词直达候选源
- 支持重复订阅判断
- 支持将已有订阅迁移到当前栏目
- 支持推荐订阅基础列表
- 支持多候选源展示与选择
- 支持跨栏目搜索结果分组
- 支持热门订阅区块
- 支持候选源/推荐源的平台标签与图标 fallback
- 支持候选排序与基础 fallback
- 支持关键词跨栏目直达候选

## 主要差距

### P0 发现链路能力

- 桌面端支持更完整的 `resolveProfileUrl` 流程
  - Harmony 目前已覆盖 YouTube、X、Instagram、Bilibili、GitHub 的基础映射
  - 也补了基础 fallback 与候选重排
  - 还缺少更细的远端探测、账号信息探针与实例级健康策略
- 桌面端支持更丰富的 discover 搜索
  - Harmony 目前只有本地推荐源检索
  - 还没有 RSSHub route 搜索、热门源搜索、平台关键词搜索
- 桌面端会更细致地区分“已订阅 / 推荐源转正 / 跨栏目迁移 / 同源不同实例”
  - Harmony 目前只做到基础重复判断与栏目迁移

### P1 展示与交互

- 桌面端候选源会带更明确的平台身份、来源类型、头像/图标
- 桌面端推荐源与 discover 结果有更强的分组和层次
- 桌面端对搜索态、空态、失败态、成功态的提示更完整

### P1 数据来源

- 桌面端有更完整的推荐源、热门源、bundle 数据
- Harmony 当前推荐源样本仍偏少，覆盖面不够

### P2 平台专项能力

- Instagram / X / YouTube / Bilibili 的专项 probe 还没迁
- 账号页解析后的更多候选策略还没迁
- 账号头像、订阅者量、平台说明等信息还没迁

## 建议补齐顺序

1. 先补展示层
   - 候选源/推荐源头像、平台标签、来源标签
   - 搜索结果层次和分组
2. 再补 discover 搜索层
   - 跨栏目推荐源搜索
   - 热门源与平台源搜索
3. 再补平台解析层
   - richer profile resolver
   - 候选排序与 fallback
4. 最后补平台专项 probe
   - Instagram / X / YouTube / Bilibili

## 当前进行中的下一步

- 补更完整的 discover 搜索源
- 迁移平台专项 probe
- 扩充热门源和推荐源数据
