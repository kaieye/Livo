# Harmony Home Inline Search Design

## Goal

为 Harmony 首页增加一个右上角搜索入口。搜索按钮位于首页顶部右侧，放在分段栏折叠后入口按钮的右边。点击后，顶部区域内联展开一个带动画的搜索框，并自动聚焦输入框拉起输入法。搜索只作用于当前分段，命中文本使用“设置 -> 外观”中的主题强调色高亮。

## Scope

In scope:

- 首页顶部右侧搜索按钮与内联展开搜索框
- 当前首页分段内的文本匹配与高亮
- 高亮颜色跟随 `ThemeService` 的 `theme.accent`
- 搜索框展开、收起、自动聚焦、自动关闭输入法
- 首页相关 ArkTS 组件拆分与回归测试

Out of scope:

- 跨文章 / 社交 / 图片 / 视频四个分段的全局搜索
- 发现页、订阅页、设置页的搜索交互
- 搜索结果单独列表页或筛选页
- 对首页数据源做额外远程检索
- 首页以外的 HDS 标题栏系统菜单重构

## User Experience

### Entry placement

- 搜索按钮只出现在首页。
- 搜索按钮位于首页顶部右侧。
- 当分段栏折叠后，右上角会同时存在“分段栏折叠入口按钮”和新的搜索按钮，搜索按钮固定放在它的右边。
- 当分段栏未折叠时，搜索按钮仍保持在顶部右侧，不参与分段栏本体布局。

### Expand behavior

- 点击搜索按钮后，不弹出独立浮层，而是在首页顶部区域内联展开搜索框。
- 搜索框从右向左展开，带透明度和轻微位移动画。
- 展开完成后自动聚焦输入框，并调用输入控制器拉起输入法。
- 搜索框出现后，首页当前列表保持原位，不切换页面、不重排成新结果页。

### Search behavior

- 搜索仅匹配当前分段的数据。
- 输入为空时，不显示任何高亮。
- 输入非空时，不过滤卡片，只高亮命中的文字片段。
- 当前分段无匹配时，列表仍保持原样，仅显示轻量提示文案。

### Collapse behavior

- 点击关闭按钮、清空并退出、或切换到非首页 Tab 时，搜索框收起。
- 收起时带与展开对应的退出动画。
- 收起后清空高亮状态，并停止输入。

## Search Scope Rules

- 文章 / 普通卡片：匹配 `feedTitle`、`title`
- 社交卡片：匹配 `displayName`、`username`、正文 `text`、引用正文 `quotedTweet.text`
- 图片卡片：匹配作者标签、来源标签、`caption`
- 视频卡片：匹配当前网格里已展示的标题文案

搜索实现采用当前页面已加载的数据，不新增独立搜索索引。

## Highlight Rules

- 高亮只作用于文本片段，不改变整张卡片的背景色或边框色。
- 高亮主色来自 `theme.accent`，确保与“设置 -> 外观 -> 颜色”联动。
- 高亮样式需要同时兼顾浅色 / 深色主题下的可读性，优先使用“强调字色 + 轻底色”组合，而不是只改一种颜色。
- 同一段文本内允许多个命中片段。
- 匹配大小写不敏感；空白输入和纯空格输入视为未搜索。

## Layout And Architecture

### Index page

- 首页状态集中放在 `entry/src/main/ets/pages/Index.ets`，负责：
  - 搜索展开 / 收起状态
  - 搜索词
  - 输入控制器
  - 当前分段搜索结果计算
  - 非首页状态下的搜索重置
- 首页顶部标题区域增加一个首页专用 builder，用于承载：
  - 原有“今日推荐”标题
  - 右上角搜索按钮
  - 展开后的搜索框

### Search UI extraction

- 将可复用但只服务首页搜索的 UI 拆到首页附近的公共组件中，避免 `Index.ets` 继续膨胀。
- 至少拆出一层可复用的“高亮文本渲染”能力，用于普通卡片、社交卡片、图片卡片、视频卡片共用命中文本绘制逻辑。

### Card integration

- 普通文章卡片改为支持高亮 `feedTitle` 与 `title`。
- `TweetEntryCard` 增加可选搜索态输入，不改变默认展示逻辑。
- `PictureEntryCard` 增加可选搜索态输入，不改变默认图片轮播与打开逻辑。
- `HomeVideoGrid` 如有标题文本，补充可选高亮渲染入口。

## Interaction Details

- 点击搜索按钮：
  - 若当前未展开，则展开搜索框并请求输入焦点
  - 若当前已展开，则仅保持焦点，不重复切换状态
- 切换首页分段：
  - 保留当前搜索词
  - 立即根据新分段内容重新计算高亮
- 切换到底部其他 Tab：
  - 自动收起搜索框
  - 清理搜索词和输入状态
- 首页从其他 Tab 返回：
  - 默认保持关闭状态，不恢复旧搜索词
- 当分段栏折叠 / 展开时：
  - 搜索按钮位置应与折叠入口按钮协同布局
  - 不覆盖分段栏主交互点击区

## Motion

- 搜索框展开 / 收起使用现有首页温和动效风格，优先复用项目中的 motion token。
- 推荐组合：
  - 宽度变化
  - 透明度变化
  - 轻微 `translateX`
- 输入框显隐动画应短促，不抢首页滚动和分段栏折叠动画的注意力。

## Testing

- 先补失败测试，再实现功能。
- 增加首页源码级测试，覆盖：
  - 首页新增搜索状态与输入控制器
  - 首页标题区域存在搜索按钮与展开搜索框
  - 搜索按钮位于折叠分段栏按钮右侧的布局约束
  - 搜索高亮颜色来源于 `theme.accent`
  - 当前分段搜索数据来源与各类卡片高亮接线
  - 搜索框展开时触发聚焦 / 输入法逻辑
  - 切换底部 Tab 时会重置首页搜索状态
- 运行受影响的 Harmony Node 回归测试。
- 如环境允许，运行 `pnpm build:harmony:debug` 验证 ArkTS 编译通过。

## Risks

- 首页当前标题栏使用 HDS Navigation 的系统材质标题栏，若把搜索 UI 错放进系统 title bar menu，容易与现有回归测试约束冲突。
- 社交卡片和图片卡片目前是独立组件，若高亮逻辑直接复制到多个组件，后续维护成本会升高。
- ArkTS 文本富文本能力如果使用方式不当，可能导致长文本截断、高亮换行或 ellipsis 表现异常。
- 自动拉起输入法依赖输入框聚焦时机，若动画与聚焦调用顺序不对，可能出现搜索框已展开但键盘未弹出的情况。

## Acceptance Criteria

- 首页右上角出现搜索按钮，并与折叠分段栏按钮并列。
- 点击搜索按钮后，搜索框以动画方式展开。
- 搜索框展开后自动获得焦点，并自动弹出输入法。
- 输入内容后，只在当前分段内匹配文字。
- 命中文本会被高亮，且高亮主色跟随“设置 -> 外观”中的主题颜色。
- 切换分段后，搜索词保留并应用到新的当前分段。
- 切换到非首页 Tab 后，搜索框关闭且搜索状态被清理。
- 不破坏现有首页分段栏折叠、底部 Tab、滚动标题材质与卡片点击行为。
